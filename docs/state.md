# State and Artifacts

Otter keeps all per-project state under `.otter/`. It is a flat directory you can delete and
rebuild from scratch — nothing under `.otter/` is a source of truth.

## Directory Layout

```
.otter/
├── state.db            # bun:sqlite — source cursors
├── compiled/           # per-model compiled SQL, mirrors models/ tree
│   └── models/…/x.sql
└── target/
    ├── manifest.json    # DAG + compiled SQL per node
    ├── run_results.json # last build summary (model execution)
    ├── test_results.json # last build summary (column tests)
    └── events.jsonl     # append-only event log (one JSON object per line)
```

- `state.db` is created on first `otter load` and is persistent across commands.
- `compiled/` and `target/` are regenerated by `otter compile` and `otter build`. `otter clean`
  removes them entirely.
- `compiled/<path>.sql` mirrors the source path under `modelsDir` (e.g.
  `models/staging/stg_users.sql` → `.otter/compiled/models/staging/stg_users.sql`). Useful for
  diffing rendered SQL between runs without parsing the manifest.

## state.db

`.otter/state.db` is a [`bun:sqlite`](https://bun.sh/docs/api/sqlite) database used by source
drivers to persist cursors for resumable extraction.

## Cursors

```sql
CREATE TABLE IF NOT EXISTS cursors (
  source_name TEXT NOT NULL,
  stream      TEXT NOT NULL,
  value       TEXT NOT NULL,
  updated_at  INTEGER NOT NULL,
  PRIMARY KEY (source_name, stream)
);
```

| Column        | Description                                            |
| ------------- | ------------------------------------------------------ |
| `source_name` | Matches `config.sources.<name>`                        |
| `stream`      | Matches the `<stream>` passed to `otter load`          |
| `value`       | Driver-defined cursor (page token, last id, timestamp) |
| `updated_at`  | `Date.now()` at write time                             |

Drivers read and write cursors through the `CursorState` handle passed to `Source.extract`. See
[sources.md](sources.md#source-interface).

## Manifest

Written by `otter compile`. Shape:

```typescript
interface Manifest {
  generated_at: string; // ISO 8601
  order: string[]; // topo order of model ids
  nodes: Record<
    string,
    {
      id: string;
      path: string; // relative to repo root
      config: {
        materialized: "view" | "table" | "incremental";
        unique_key?: string;
        tags?: string[];
      };
      sql: string; // original tagged template output
      deps: string[]; // model ids from ref()
      sources: string[]; // "source.stream" strings from source()
      seeds: string[]; // seed names from seed()
      compiled_sql: string; // same as `sql` today; reserved for future rewrites
    }
  >;
}
```

Used by `otter build` (load manifest → re-build DAG → run) and `otter show` (look up node id).

## Run Results

Written at the end of `otter build`:

```typescript
interface RunResults {
  nodes: Record<
    string,
    {
      status: "success" | "error";
      duration_ms: number;
      error?: string;
    }
  >;
}
```

One entry per executed node. Nodes skipped by a selector are omitted. A single failure
short-circuits the build (the runner re-throws), so later nodes may be absent.

## Test Results

Written at the end of `otter build` after model execution succeeds, capturing every column test
declared via `{{ config(columns: { ... }) }}`:

```typescript
interface TestResults {
  tests: Array<{
    model: string;
    column: string;
    test: "unique" | "not_null";
    status: "pass" | "fail" | "error";
    duration_ms: number;
    failures?: number; // row count when status is "fail"
    error?: string; // error message when status is "error"
  }>;
}
```

If any test is `fail` or `error`, `otter build` exits non-zero. See
[models.md](models.md#column-tests).

## Events JSONL

Newline-delimited JSON. One object per event:

```json
{"type":"node.start","id":"stg_users","ts":"2026-04-22T03:14:07.123Z"}
{"type":"node.finish","id":"stg_users","ts":"2026-04-22T03:14:07.456Z","duration_ms":333}
{"type":"node.error","id":"fct_events","ts":"2026-04-22T03:14:07.789Z","duration_ms":120,"error":"…"}
```

Produced by `OtterEmitter` + `jsonlAppender`. See [observability.md](observability.md#emitter-api)
for the programmatic API.

## Retention and Cleanup

- `otter clean` removes `.otter/target/` and `.otter/compiled/` (manifest, run results, events
  log, per-model compiled SQL) but preserves `.otter/state.db` so source cursors survive.
- Delete `.otter/` entirely to forget everything (cursors too). Next `otter load` will re-create
  an empty `state.db`.
